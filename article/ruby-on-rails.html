Ruby on Rails

Ruby on Rails, also known as RoR and Rails, is a web framework used for web development. Like most application frameworks, RoR was built with the Ruby programming language for a specific task that its creator had while working at a web application development company. That's why Ruby on Rails is often referred to as "startup technology" because it was built for quick launches.

What is the difference between Ruby and Ruby on Rails?

The first thing you should notice immediately is dependency. Ruby on Rails depends on Ruby to exist and function. On the other hand, Ruby does not require Rails to function and is often used without the additional use of Rails. Simply put, Ruby on Rails is an enhancement or add-on to Ruby.

Ruby is a general-purpose programming language, as are C #, C ++, Java, JavaScript, Python, Ruby, and Visual Basic. Ruby has its own unique syntax and rules, since the job of a programming language is to tell the computer what to do and how to do it.

On the other hand, Rails is not a programming language, it is a web framework or web development framework. Web application frameworks like Rails still use scripts written by developers, but they do not use their own language. Rails uses the Ruby programming language.

The Rails philosophy is based on these two principles:

DRY ("Don't Repeat Yourself") - suggests that writing the same code over and over is bad practice.

"Configuration Convention" - means that Rails makes some assumptions about what you want to do and how you are going to do it, rather than requiring you to specify every little thing through endless configuration files.
